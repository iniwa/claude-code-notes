セットアップワークフロー
新規プロジェクト開始から実装まで、この手順に沿って進める。
1
webチャットで設計書を作成する
テンプレートを元にClaudeと相談しながら plan.md を作成する。
同時に CLAUDE.md の草案も作る。設計時に決まった技術スタック・禁止事項・コーディング規約をここに書いておくと、CLIでの毎セッションの説明コストがゼロになる。

成果物：plan.md  /  CLAUDE.md（草案）
2
CLAUDE.md と plan.md を Claude-startup で再編集する
ステップ1で作成した CLAUDE.md と plan.md を、Claude-startup（専用プロジェクト）で再編集する。

webチャットでの設計段階では含められない CLI実行環境固有の情報（PCスペック、Docker構成、OS設定、パス情報など）を、ベーステンプレート（CLAUDEmd_docker / CLAUDEmd_windows）から組み込む。
これにより、Claude Code が実際の実行環境を正しく認識した状態でコーディングを開始できる。

※ このプロジェクトのリポジトリ構成は後日調整予定。
3
リポジトリを用意する
新規の場合はディレクトリを作成してgitを初期化。既存プロジェクトは git clone。 
mkdir my-project && cd my-project
git init && git commit --allow-empty -m "init"
git を最初に初期化しておくことで、Claude Code の変更を git diff で追跡・revert できるようになる。
大きな変更を依頼する前は 必ずブランチを切ること。
4
プロジェクトファイルを設置する
以下のファイルをプロジェクトルートに配置する。テンプレートは「テンプレート集」を参照。
my-project/
├── CLAUDE.md  # Claude への常設指示（技術スタック・規約等）
├── plan.md  # 設計書・実装フェーズ
├── .claudeignore  # node_modules/ 等を除外（必須）
├── .claude/
│   └── settings.json  # 権限・フック設定
└── src/  ...
.claudeignore は必ず作成する。
node_modules/, .git/, build/, dist/ 等を除外することで、Claude Code が読み込むコンテキストを減らしトークン消費を抑える。
5
VS Code で開き、Claude Code を起動する
VS Code でプロジェクトルートを開き、統合ターミナルで Claude Code を起動する。 
# モデルを明示指定して起動
claude --model sonnet

# または重いタスクなら
claude --model opus
Claude Code for VS Code 拡張を使うと、ファイルの差分確認がエディタ上でできるようになる。
6
実装を依頼する
ファイルを明示参照して指示する。「見ながら」より @ファイル名 で参照する方が確実。 
@plan.md のPhase 1 を実装してください
大きなタスクの前は Shift+Tab でプランモードに切り替える。
何をするか計画を確認してから承認することで、意図しない変更を防ぐ。

タスクは細かく分割して渡す。「全部実装して」より「まず認証部分だけ」の方が精度が上がる。
7
デバッグ・修正を繰り返す
エラーはコピペせず、! コマンドでターミナル出力をそのまま Claude に渡す。 
! npm test 2>&1
! npm run build 2>&1 | tail -30
コンテキスト管理のルール：
使用率 80% 前後 → /compact で圧縮（作業継続）
別タスクへ切り替え → /clear で完全リセット

フックを設定しておくと、ファイル保存のたびに lint が自動実行されエラーを早期検知できる（→ テンプレート集 参照）。
8
動作確認・コミット
動作確認後、git commit してプッシュ。コミットメッセージの生成も Claude に任せられる。 
git diff --staged | claude -p "コミットメッセージを日本語で生成して"
Bash(git commit:*) を permissions.allow に入れておくと、Claude がコミットまで自動化できる。
