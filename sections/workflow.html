併用ワークフロー
Codex で意図を作り、Claude Code に渡し、Codex で受け止める。1 タスクの定型ループ。
基本姿勢

役割は固定しない。曖昧さが高いほど Codex、実装が明確で量があるほど Claude Code。両者の間に docs/handoffs/YYYY-MM-DD-<task>.md を置き、そこを介してやり取りする。

1
Codex で依頼を分類する
変更種別（文言修正 / 既存調整 / 新規 / 設計変更 / ドキュメント）と影響範囲を最初に切り分ける。 
この依頼の変更種別、影響範囲、設計リスク、
Codex で続行するか Claude Code に handoff するかを分類してください。
2
Codex で設計リスクを切る
責務境界、公開 API、永続化、デプロイに触れるかを Codex で確認する。
高リスクなら先に判断を AGENTS.md に追記する。低リスクなら追記不要。
3
Claude Code に渡せる粒度まで絞る
Goal、Files To Inspect、Files To Edit、Constraints、Non Goals、Verification、Expected Report がそろうまで Codex で詰める。揃わなければ渡さない。
4
docs/handoffs/YYYY-MM-DD-<task>.md を作る
Codex が handoff ファイルを書き、ユーザーにパスを伝える。
ユーザーは Claude Code にそのパスを渡す。 
@docs/handoffs/2026-05-08-xxx.md を確認し、
このプロジェクトの調整を行ってください。
5
Claude Code が実装・検証・報告する
Claude Code は AGENTS.md、CLAUDE.md、handoff を読み、Files To Edit の中だけで作業する。
範囲外が必要になったら止まって質問する。
6
Codex が差分をレビューする
handoff 外の変更、不要なパターン追加、検証不足、公開不可情報の混入がないかを確認する。 
以下の Claude Code 報告と差分を、
AGENTS.md と handoff に照らしてレビューしてください。
7
必要な判断だけ AGENTS.md に残す
今後も守るべき制約・責務境界・採用根拠が出たときだけ追記する。
単純な文言修正、CSS 微調整、既存方針に沿った小変更では追記しない。
