ステータスバー設定
Codex / Claude Code のステータスラインにモデル名・コンテキスト使用率・Git情報等を表示するカスタマイズ手順。

概要

Claude Code は任意スクリプト方式、Codex は組み込み項目の選択方式でステータスラインを設定できる。Claude Code は ~/.claude/statusline.py を呼び出せるため自由度が高く、Codex は ~/.codex/config.toml の status_line に表示項目を並べる。

Codex のステータスライン

Codex CLI では ~/.codex/config.toml に次のように追加する。設定後は新しい Codex セッションから反映される。 
status_line = ["model-with-reasoning", "fast-mode", "git-branch", "current-dir", "context-remaining"]
表示例: 
gpt-5.5 medium · fast · main · D:\Git\ai-coding-notes · Context 42% left
TUI 内では /statusline から表示項目を切り替えられる。Claude Code のように任意スクリプトを実行する方式ではないため、レート制限・セッションコスト・入出力トークンの独自集計は標準の status line だけでは扱わない。

Codex で使う主な項目

model-with-reasoning
モデル名と reasoning effort を表示する。
fast-mode
Fast mode が有効なときに表示する。
git-branch
現在の Git ブランチを表示する。
current-dir
現在の作業ディレクトリを表示する。
context-remaining
残りコンテキスト割合を表示する。

Claude Code のセットアップ方法

以下のプロンプトを Claude Code の CLI にそのまま貼り付けて実行してください。自動でファイル作成と設定が行われます。
Claude Code に貼り付けるプロンプト
ステータスバーをセットアップしてください。以下の仕様で実装してください。

■ 表示例
🤖 Sonnet 4.6 │ 📊 54% ctx │ 💰 $0.620 │ 🔀 main │ 🎯 1.2k↑ 5.7k↓
⏱ 5h  ▰▰▰▱▱▱▱▱▱▱  30%  Resets 1pm (Asia/Tokyo)
📅 7d  ▰▰▰▰▰▰▰▰▰▰  97%  Resets Mar 7 at 5pm (Asia/Tokyo)

■ 構成
- ~/.claude/statusline.py — メインスクリプト（Python）
- ~/.claude/statusline-command.sh — 起動用シェルスクリプト
- ~/.claude/settings.json に statusLine 設定を追記

■ 表示内容（1行目）
モデル名 | コンテキスト使用率 | セッションコスト | Git差分(+X/-Y) | Gitブランチ | トークン数(入力↑ 出力↓)

■ 表示内容（2行目・3行目）レート制限
- 5時間レート: プログレスバー(10段階) + パーセント + リセット時刻
- 7日レート: プログレスバー(10段階) + パーセント + リセット日時
- タイムゾーンは自分の環境に合わせること

■ レート制限の取得方法
- ~/.claude/.credentials.json から OAuth トークンを読み取り
- claude-haiku-4-5-20251001 へ最小限のAPIコール(max_tokens:1)を送信
- レスポンスヘッダの anthropic-ratelimit-unified-* を解析
- 結果は6分間キャッシュ（$TMPDIR/claude-ratelimit-cache.json）

■ 色分け
- 0-49%: 緑 / 50-79%: 黄 / 80-100%: 赤

■ 注意
- Windows環境の場合、UTF-8出力とCRLF問題に対応すること
- stdin から JSON を受け取り、stdout に表示行を出力する形式

トークン消費量

レート制限の取得に claude-haiku-4-5-20251001 への API コールが発生します。
項目詳細
1回あたり入力 ~30 トークン / 出力 1 トークン（最小リクエスト）
キャッシュ間隔6分（アクティブに使用中のみ）
1時間あたり最大 ~10回 = 入力 ~300 トークン / 出力 ~10 トークン
コスト影響Haiku の最小リクエストのため、金銭的コストはほぼゼロ
レート制限への影響ごく僅か（Haiku のレートは Opus/Sonnet と統合上限に含まれるが、消費量は無視できるレベル）

※ レート制限表示が不要であれば、該当部分を省略するよう指示することで API コールを完全になくすことも可能です。

前提条件

  • python または python3 がコマンドラインから実行可能であること
  • Claude Code に OAuth ログイン済みであること（レート制限表示に必要。~/.claude/.credentials.json が生成される）
  • レート制限表示は Pro / Max サブスクリプションの OAuth 認証が前提。API キー利用者はレート制限部分を省略してください

カスタマイズのヒント

  • 表示項目の追加・削除はプロンプトの「表示内容」を編集するだけ。例: コスト表示が不要なら該当行を削除
  • キャッシュ間隔を変更したい場合は「キャッシュ時間を X 分に変更」と追記
  • キャッシュが壊れた場合は $TMPDIR/claude-ratelimit-cache.json を削除すれば再取得される
