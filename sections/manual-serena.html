Serena マニュアル
Serena MCP の基本的な使い方。自然言語で指示するだけで、裏側で LSP ベースのツールが自動的に動く。

基本ワークフロー

Serena の最大の特徴は、特別なコマンドを覚える必要がないこと。普段通り Codex / Claude Code に自然言語でお願いするだけで、エージェントが裏側で Serena のツール（LSP）を自動操作する。

Codex で使い始める

Codex の ~/.codex/config.toml に mcp_servers.serena を登録したら、新しい Codex セッションを開く。作業開始時に必要なら次のように指示する。 
Serena で現在のディレクトリをプロジェクトとして有効化してください
このリポジトリのように .serena/project.yml がある場合、--project-from-cwd により現在の作業ディレクトリからプロジェクトを自動検出できる。

1. 構造的な検索

find_symbol
「〇〇の処理をしている関数の定義を見つけて」
ファイル全体を読まずにピンポイントで関数定義を探す。
find_referencing_symbols
「この変数がプロジェクト内のどこから呼び出されているか調べて」
参照元を構造的に追跡する。

2. 全体像を把握

get_symbols_overview
「このファイルの全体的なシンボル（クラスや関数）の概要を教えて」
コードの目次だけを素早く取得する。

3. 安全にコードを編集

replace_symbol_body
「〇〇という関数の内部だけを、この新しい処理に書き換えて」
行数のズレを気にせず「関数というブロック単位」で安全にコードを差し替える。

Web ダッシュボード

Serena の解析ログや記憶データを視覚的に確認できる。

手順:
1. Codex / Claude Code を起動した状態にする（Serena サーバーも同時に起動）
2. ブラウザで http://localhost:24282/dashboard/index.html または http://localhost:8000 を開く

クライアントや起動方法によってポートが変わる。表示されない場合は Serena のログや Codex の codex mcp list を確認する。
