IDE 連携
VS Code・JetBrains での Claude Code 拡張の使い方と、CLI との違い。

利用可能なインターフェース

CLI（ターミナル）メイン
最もフル機能。MCP・フック・自動化フラグすべてに対応。ターミナルで claude を実行するだけで起動。
VS Code 拡張推奨
エディタ内で Claude Code を使える。ファイルの差分をエディタのdiffビューで確認でき、インライン編集が直感的。
JetBrains 拡張
IntelliJ IDEA、PyCharm、WebStorm 等の JetBrains IDE に対応。VS Code 版と同等の機能を提供。
デスクトップアプリ
Mac / Windows 向けのスタンドアロンアプリ。エディタを使わない場面で便利。
Web（claude.ai/code）新機能
ブラウザから Claude Code を利用。ローカル環境のセットアップなしで使える。

VS Code 拡張の主要機能

Diff ビュー便利
Claude がファイルを編集する際、VS Code のネイティブ diff ビューで変更内容を確認できる。CLI では文字ベースの差分表示だが、VS Code ではエディタ内でカラフルな差分を視覚的にレビュー可能。
インライン編集
エディタ上でコードを選択し、右クリックまたはコマンドパレットから Claude に編集を依頼できる。選択範囲がそのままコンテキストとして渡される。
ファイル参照
チャットパネルで @ファイル名 を使ってファイルを参照。VS Code のファイルエクスプローラからドラッグ＆ドロップでも参照を追加できる。
ターミナル統合
VS Code の統合ターミナルで CLI 版 Claude Code を使えば、エディタと CLI の両方の利点を活かせる。拡張のGUIパネルとターミナルCLIは同時に使用可能。

VS Code セットアップ

1. 拡張機能のインストール 
# VS Code の拡張機能マーケットプレイスで検索
Claude Code
2. 認証
拡張機能を開くと自動的に認証フローが始まる。CLI で既にログイン済みなら認証情報が共有される。

3. 使い方
サイドバーの Claude アイコンをクリックしてチャットパネルを開く。または Ctrl+Shift+P → 「Claude」で各コマンドにアクセス。

CLI vs IDE 拡張の使い分け

CLI が向いている場面：自動化（-p フラグ）、CI/CD 統合、MCP サーバー管理、フック設定、ヘッドレス環境

IDE 拡張が向いている場面：日常的なコーディング、diff レビュー、インライン編集、ファイルのドラッグ＆ドロップ参照

おすすめ：VS Code + 統合ターミナルで CLI を併用する構成。エディタの視覚的なメリットと CLI のフル機能を両立できる。
