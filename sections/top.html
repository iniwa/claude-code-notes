⚠ ご注意

このサイトは いにわ（iniwa） 本人が AI コーディングを進めるための考え方、手順、テンプレート例をまとめた公開用リファレンスです。

個人環境に強く依存する詳細テンプレートや認証情報は含めず、公開しても差し支えない汎用的な内容に整理しています。実運用では各自の環境に合わせて読み替えてください。

個人用の詳細テンプレートは別リポジトリ・別ディレクトリで管理する前提です。

このサイトについて

Codex と Claude Code を併用するためのワークフロー、設計思想の蓄積方法、実装ハンドオフ、Claude Code の詳細設定を一つの SPA にまとめています。

  • 全体ワークフロー — Codex で設計し、Claude Code で実装し、Codex で設計整合性を確認する流れ
  • Codex 詳細 — 設計思想、意思決定ログ、実装ハンドオフ、レビュー観点の蓄積
  • Claude Code 詳細 — CLI、設定、MCP、Hooks、IDE、サブエージェントなどの実装リファレンス
  • テンプレート — AGENTS.md、CLAUDE.md、plan.md、設定ファイルの公開用サンプル
基本方針

Codex は設計判断の継続性を守る場所として使う。なぜその設計にしたのか、何を変えてはいけないのか、実装後にどこを確認すべきかを整理する。

Claude Code は実装実行環境として使う。Codex が整理したハンドオフと CLAUDE.md の作業規約に従って、編集・検証・差分整理を進める。
