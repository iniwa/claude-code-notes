個人用の詳細テンプレートは別リポジトリ・別ディレクトリで管理する前提です。

このサイトについて

Codex と Claude Code を併用するための判断基準、設計思想の蓄積方法、実装ハンドオフ、Claude Code の詳細設定を一つの SPA にまとめています。

  • 全体ワークフロー - 変更の曖昧さ、設計リスク、実行量に応じて担当を切り替える流れ
  • Codex 詳細 - プロジェクト初期設定、作業判定、設計判断、handoff、レビュー、記録更新を扱う運用
  • Claude Code 詳細 - CLI、設定、MCP、Hooks、IDE、サブエージェントなどの実装実行リファレンス
  • テンプレート - AGENTS.md、CLAUDE.md、plan.md、設定ファイルの公開用サンプル
現在の基本方針

Codex は、曖昧な要求を整理し、設計判断を記録し、必要ならそのまま実装・検証・レビューまで進める作業場として使う。単なる設計係ではなく、設計リスクを見ながら差分を閉じるための主担当にする。

Claude Code は、実装方針が決まった後の実行力、CLI 連携、Hooks、サブエージェント、自動化を活かす場面で使う。明確な handoff と CLAUDE.md の作業規約に従って、編集・検証・差分整理を進める。

つまり「Codex = 設計、Claude Code = 実装」と固定しない。曖昧さが高いほど Codex、実装タスクが明確で量が多いほど Claude Code を優先する。
