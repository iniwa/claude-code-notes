CI/CD レシピ集
GitHub Actions で Claude Code を活用する実践的なレシピ。PR レビュー自動化、コード品質チェック、ドキュメント生成など。

前提条件

CI/CD で Claude Code を使うには以下が必要：
  • ANTHROPIC_API_KEY を GitHub Secrets に登録
  • Claude Code を CI 環境にインストール（npm install -g @anthropic-ai/claude-code）
  • 非インタラクティブモード（-p フラグ）で実行
  • コンテナ環境では --dangerously-skip-permissions を使用可能

PR レビュー自動化

pr-review.ymlおすすめ
PR が作成されたら自動で Claude にコードレビューを依頼する。 
name: Claude PR Review
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Install Claude Code
        run: npm install -g @anthropic-ai/claude-code

      - name: Review PR
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          git diff origin/main...HEAD | claude -p \
            "このdiffをレビューしてください。
             セキュリティ・パフォーマンス・可読性の観点で
             問題点があれば指摘してください。" \
            --allowedTools "Read,Grep,Glob" \
            --output-format json | jq -r '.result' > review.md

      - name: Post Review Comment
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const review = fs.readFileSync('review.md', 'utf8');
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: '## 🤖 Claude Code Review\n\n' + review
            });

コミットメッセージ生成

ワンショット
ステージ済みの変更からコミットメッセージを自動生成する。ローカルでも CI でも使える。 
# ローカルで使う場合
git diff --staged | claude -p \
  "このdiffに適切なコミットメッセージを日本語で生成してください。
   Conventional Commits形式で。本文のみ出力。"

# git alias に登録
# ~/.gitconfig
[alias]
  ai-commit = "!git diff --staged | claude -p 'コミットメッセージを日本語で生成。Conventional Commits形式。本文のみ出力。' | git commit -F -"

コード品質チェック

quality-check.yml
PR の変更ファイルに対してセキュリティ・品質チェックを実行する。 
- name: Security Check
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
  run: |
    claude -p \
      "このリポジトリの変更ファイルについて、
       OWASP Top 10の観点でセキュリティチェックしてください。
       問題がなければ'問題なし'とだけ返してください。" \
      --allowedTools "Read,Grep,Glob" \
      --max-turns 5

ドキュメント自動生成

doc-gen.yml
コード変更時に API ドキュメントや CHANGELOG を自動更新する。 
- name: Update Docs
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
  run: |
    claude -p \
      "git diff origin/main...HEAD を確認し、
       公開APIに変更があればCHANGELOG.mdを更新してください。
       変更がなければ何もしないでください。" \
      --allowedTools "Read,Write,Edit,Bash(git diff*),Grep,Glob" \
      --max-turns 10

注意事項

コスト管理：--max-turns を必ず設定し、無限ループを防止する。PR レビューなら 3-5 ターン、コード修正を伴うタスクなら 10-20 ターンが目安。

セキュリティ：--dangerously-skip-permissions はコンテナ内でのみ使う。--allowedTools でツールを最小限に絞る。

モデル選択：CI ではコスト重視で --model sonnet を推奨。複雑な分析が必要な場合のみ Opus を使う。
