/config — 権限・セキュリティ
ツール使用の許可・拒否ルール。プロジェクトの .claude/settings.json に書いてgit管理するのが基本。
評価順序： deny → ask → allow の順に評価され、最初にマッチしたルールが適用される。denyを先に書けば確実にブロックできる。
permissions.defaultMode両方おすすめ
デフォルトの権限モード。"default"（毎回確認）/ "acceptEdits"（編集自動承認）/ "bypassPermissions"（全スキップ・危険） 
"permissions": { "defaultMode": "acceptEdits" }
permissions.allow両方おすすめ
確認なしで許可するツール・コマンドの一覧。ワイルドカード対応。 
"allow": [
  "Bash(npm run *)",
  "Bash(git log *)",
  "Bash(git diff *)",
  "Bash(git status)"
]
permissions.deny両方おすすめ
常に拒否するツール・コマンドの一覧。denyが最初に評価されるため確実にブロックできる。 
"deny": [
  "Bash(rm -rf *)",
  "Bash(curl *)",
  "Read(.env)",
  "Read(.env.*)"
]
filesystem.denyRead両方
読み取りを禁止するパスの一覧。秘密鍵や認証情報ファイルを保護する用途。 
"filesystem": {
  "denyRead": ["~/.ssh/*", "~/.aws/*", "~/.gnupg/*"]
}
allow パターン構文両方おすすめ
* ワイルドカードでツール名とパラメータの両方をパターンマッチできる。 
"allow": [
  "Bash(git:*)",          // git で始まる全コマンド
  "Bash(npm test:*)",     // npm test 系を許可
  "Bash(dir:*)",          // dir コマンド全般
  "Read(/path/to/**)",    // 特定パス配下の読み取り
  "WebFetch(domain:docs.anthropic.com)"  // 特定ドメイン限定
]
MCP ツール一括許可両方おすすめ
MCP ツールの命名規則は mcp__<サーバー名>__<ツール名>。* でサーバー単位の一括許可が可能。 
"allow": [
  "mcp__serena__*",       // Serena 全ツール
  "mcp__tavily__*",       // Tavily 全ツール
  "mcp__playwright__*",   // Playwright 全ツール
  "mcp__git__*",          // Git MCP 全ツール
  "mcp__*"                // 全MCP の全ツール（注意）
]
個別指定も可："mcp__serena__find_symbol" のように1つずつ指定できる。セッション中に手動許可した操作は settings.local.json に自動追加される。
allow + deny 組み合わせ両方
deny は allow より常に優先される。「広く許可して一部をブロック」のパターンが実用的。 
"permissions": {
  "allow": [
    "Bash(git:*)",
    "mcp__serena__*"
  ],
  "deny": [
    "Bash(git push --force:*)",
    "Bash(git reset --hard:*)"
  ]
}
プロジェクト設定の例プロジェクト
.claude/settings.json — チームで共有。git にコミットして統一ルールを適用する。 
// .claude/settings.json（チーム共有）
{
  "permissions": {
    "allow": [
      "Bash(npm run *)",
      "Bash(git status:*)",
      "Bash(git diff:*)",
      "Bash(git log:*)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Read(.env)",
      "Read(.env.*)"
    ]
  }
}
ユーザー設定の例ユーザー
~/.claude/settings.json — 全プロジェクト共通の個人設定。よく使う MCP やコマンドをここで許可しておくと効率的。 
// ~/.claude/settings.json（個人・全プロジェクト共通）
{
  "permissions": {
    "allow": [
      "mcp__serena__*",
      "mcp__tavily__*",
      "Bash(git:*)",
      "Bash(dir:*)"
    ],
    "deny": [
      "Bash(git push --force:*)",
      "Bash(git reset --hard:*)"
    ]
  }
}
ローカル設定の例ローカル
.claude/settings.local.json — 個人のプロジェクト固有設定。.gitignore に入れてチームと共有しない。セッション中の手動許可はここに自動蓄積される。 
// .claude/settings.local.json（個人・非共有）
{
  "permissions": {
    "allow": [
      "mcp__serena__activate_project",
      "mcp__serena__onboarding",
      "Bash(git push)"
    ]
  }
}
enableAllProjectMcpServers両方
プロジェクトの .mcp.json を自動的に全て有効化。デフォルト false。gitリポジトリ経由の悪意あるMCPサーバー対策のため無闘に有効にしない。 
"enableAllProjectMcpServers": false
attribution.coAuthoredBy両方
gitコミットメッセージに付加されるCo-authored-by表記の制御。空文字列で非表示にできる。 
"attribution": { "coAuthoredBy": "" }
設定マージの仕組み： 全スコープ（User / Project / Local / Managed）の allow と deny はマージされる。いずれかのスコープで allow されていれば許可。deny はどのスコープで書いても最優先でブロック。詳細は「スコープ構造」セクションを参照。
