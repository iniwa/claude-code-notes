/config — MCP・拡張
Model Context Protocolサーバーを使って Codex / Claude Code にツールを追加拡張する設定。
Codex: mcp_serversユーザー
Codex のグローバル MCP は ~/.codex/config.toml に保存する。基本セットは Serena / Playwright / Git / Tavily。 
[mcp_servers.serena]
startup_timeout_sec = 15
command = "uvx"
args = ["--from", "git+https://github.com/oraios/serena", "serena", "start-mcp-server", "--project-from-cwd", "--context=codex"]

[mcp_servers.playwright]
command = "npx"
args = ["-y", "@playwright/mcp@latest"]

[mcp_servers.git]
command = "uvx"
args = ["mcp-server-git"]

[mcp_servers.tavily]
command = "npx"
args = ["-y", "tavily-mcp"]
Claude Code: mcpServers両方
Claude Code の MCP 定義。プロジェクト共有は .mcp.json、個人用はグローバル設定に記述する。CLI では --scope user を付ける。 
"mcpServers": {
  "playwright": {
    "command": "npx",
    "args": ["-y", "@playwright/mcp@latest"]
  },
  "git": {
    "command": "uvx",
    "args": ["mcp-server-git"]
  },
  "tavily": {
    "command": "npx",
    "args": ["-y", "tavily-mcp"]
  }
}
env（settings.json内）両方おすすめ
Tavily などのAPIキーは設定ファイルに直書きせず、OS のユーザー環境変数で持つ。Windows では次の形で永続化する。 
[Environment]::SetEnvironmentVariable("TAVILY_API_KEY", "tvly-...", "User")
反映には Codex / Claude Code / ターミナルの再起動が必要。
allowedMcpServers両方
使用を許可するMCPサーバーのホワイトリスト。チームでのセキュリティ管理用。未設定時は全サーバーが対象。
channelsEnabledユーザーTeam/Enterprise
チャンネル機能を有効化。--channels フラグと組み合わせることでツール承認プロンプトをスマホに転送するなどができる。
