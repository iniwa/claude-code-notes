CLIフラグ — 自動化モード
BOT・CI/CD・ラズパイ等のヘッドレス環境での利用に必須のフラグ群。
-p "プロンプト"自動化よく使う
ノンインタラクティブモード（旧称 headless mode）。プロンプトを渡して応答を返したら終了。BOT・CI/CDの基本。 
claude -p "auth.pyのバグを修正して" --allowedTools "Read,Edit,Bash"
# パイプでも使える
cat error.log | claude -p "このエラーの原因を診断して"
--output-format json自動化
出力をJSON形式に。プログラムから処理しやすくなる。stream-json はリアルタイムストリーミング処理用。 
claude -p "TODOを一覧化" --output-format json | jq '.result'
--allowedTools "..."自動化よく使う
使用許可するツールを明示指定。非インタラクティブ時は必須。ワイルドカード対応。 
--allowedTools "Read,Grep,Glob"          # 読み取り専用（安全）
--allowedTools "Read,Write,Edit"         # 書き込みあり
--allowedTools "Bash(git *),Read,Edit"   # git操作のみ許可
--max-turns N自動化
最大ターン数を制限。無限ループ防止に重要。--max-turns 1 で一問一答、--max-turns 10 で複数ステップのタスク向け。
--continue / --resume [ID]自動化
会話を継続。--continue は直近のセッション、--resume はセッションIDで特定の過去セッションを再開。 
claude -p "レビューして" --output-format json > review.json
claude -p "データベースのクエリに絞って" --continue
--bare自動化
起動時の自動検出（hooks・MCP・CLAUDE.md等）をスキップ。起動を高速化したい自動化環境向け。ANTHROPIC_API_KEY または apiKeyHelper が必要。
--dangerously-skip-permissions危険
全権限確認をスキップ。コンテナ・CI隔離環境専用。本番・手元マシンでは絶対に使わない。
