Codex 実作業ループ
Codex で指示を作り、Claude Code で実作業し、Codex でレビューして記録する流れ。

全体の流れ

1
Codex が文脈を読む
依頼内容、AGENTS.md、CLAUDE.md、関連ファイル、既存ドキュメントを確認する。
2
Codex が handoff を作る
目的、背景、編集範囲、制約、非目標、検証、報告形式を明確にして Claude Code に渡せる形にする。
3
Claude Code が実装する
Claude Code は handoff と CLAUDE.md に従い、範囲外の変更が必要なら止まって確認する。
4
Claude Code が報告する
変更ファイル、概要、検証結果、未実行の検証、Codex に戻すべき質問を返す。
5
Codex がレビューして記録する
差分が handoff と設計意図に合うか確認し、必要な判断だけ AGENTS.md や docs/ に残す。

Claude Code に貼る形式

codex-to-claude.md
AGENTS.md と CLAUDE.md を読んだうえで、以下の handoff に従って実装してください。
設計制約から外れる必要がある場合、または編集範囲が handoff と合わない場合は、実装前に止まって確認してください。
