Codex 実作業ループ
実際の会話の往復。何が誰の手元から誰に渡るかを把握する。

成果物の流れ

user request
  -> Codex: 分類・設計判断・handoff 生成
       (artifact: docs/handoffs/YYYY-MM-DD-task.md)
  -> Claude Code: 実装・検証・報告
       (artifact: changed files + report)
  -> Codex: 差分レビュー
  -> Codex: 必要なら AGENTS.md に判断を追記

ステップごとの中身

1
Codex が文脈を読む
依頼内容、AGENTS.md、CLAUDE.md、関連ファイル、既存ドキュメントを確認する。
2
Codex が handoff を作る
Goal / Background / Files / Constraints / Non Goals / Verification / Expected Report を埋め切ってからファイルに保存する。
3
Claude Code が実装する
Claude Code は handoff と CLAUDE.md に従い、Files To Edit の外が必要になったら止まって質問する。
4
Claude Code が報告する
変更ファイル、サマリ、検証結果、blocked checks、Codex に戻すべき設計質問を返す。
5
Codex がレビューして必要なら記録する
差分が handoff と設計意図に合うか確認し、今後も守るべき判断だけを AGENTS.md に追記する。

Claude Code に渡す依頼

prompt
@docs/handoffs/YYYY-MM-DD-<task>.md を確認し、
このプロジェクトの調整を行ってください。

## Verification

- python -m http.server 8787 --bind 127.0.0.1 でローカル配信し、

## Blocked Checks
- ブラウザでの目視確認はこの環境ではできなかった。

## Files Edited Outside Files To Edit
- なし

## Design Questions For Codex
- 判定マトリクスに「設定ファイル変更」行も追加すべきか。

Codex 側のレビュー依頼

review-prompt
以下は Claude Code の実装報告と差分です。
AGENTS.md と handoff に照らしてレビューしてください。

確認したいこと:
- handoff 外の変更がないか
- 制約と非目標が守られているか
- 公開不可情報が混ざっていないか
- 検証が十分か
- AGENTS.md に残すべき新しい判断があるか
