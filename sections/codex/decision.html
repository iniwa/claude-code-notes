作業判定と分岐
状況ごとに Codex で閉じるか Claude Code に渡すかを切り分ける。

判定マトリクス

状況 Codex でやる Claude Code へ渡す 注意点
文言修正 そのまま編集 多数の文言を機械的に直す場合のみ 設計判断は伴わない。AGENTS.md 追記不要。
新規セクション追加 1 セクションで構造が新しい場合 テンプレに沿って数セクション足す場合 nav・SECTIONS・index.html プレースホルダの 3 点更新を必ず指示。
複数ファイルの機械的編集 2〜3 ファイルなら直接 同じパターンを 4 ファイル以上に適用するなら handoff 編集パターンと適用範囲を Constraints に明記する。
設計変更 判断と AGENTS.md 更新は Codex で完結 判断確定後の機械的反映だけ 設計判断と実装を 1 つの handoff に混ぜない。
バグ修正 原因特定と局所修正が同時に進む場合 原因が特定済みで修正範囲が広い場合 handoff には Non Goals に「リファクタしない」を入れる。
テンプレート更新 templates/ 1〜2 ファイルの調整 テンプレと参照セクション両方を更新する場合 公開可否を Codex で必ずレビューする。

Codex に残すか Claude Code に渡すか

Codex で続行:
- 要求が曖昧
- 設計意図がまだ決まっていない
- 責務境界が動く可能性がある
- 変更範囲が小さく、レビューしながら閉じられる

Claude Code に渡す:
- Goal / Files / Constraints / Verification が明確
- 実装や機械的編集が中心
- ファイル数が多い
- 同じパターンを反復適用する

設計リスクの見方

新しい責務が増えるか
既存 API やデータ構造を変えるか
UI 層とドメイン層の境界を動かすか
永続化、認証、デプロイに触れるか
一時対応が今後の標準になりそうか
他の実装者が同じ判断を再利用しそうか

handoff できる状態

実装者が迷わず編集できるところまで Codex で詰めてから渡す。
Goal、Background、Files To Inspect、Files To Edit、Constraints、Non Goals、Verification、Expected Report がそろっていない handoff は渡さない。
