Codex プロジェクトセットアップ
新しいプロジェクトで Codex と Claude Code を併用するための初期配置と役割分担。

最初に置くファイル

AGENTS.md
Codex 側の設計意図、handoff ルール、レビュー観点を記録する。作業の判断基準を残す場所。
CLAUDE.md
Claude Code 側の実行規約を記録する。編集範囲、検証、停止条件、報告形式を書く。
.claudeignore
Claude Code に読ませないファイルを除外する。機密情報、大容量ファイル、生成物は必ず外す。
docs/
再利用できる設計判断、運用手順、調査結果を残す。会話リセット後も引き継ぐ知識置き場。

CLAUDEmdStrage から作る

新規プロジェクトでは、まず D:\Git\CLAUDEmdStrage\_base のベースを参照する。
Windows / ローカルツールでは CLAUDE_windows.md、Raspberry Pi Docker ツールでは CLAUDE_docker.md を使い、Codex 側の基準として AGENTS.md を合わせて作る。
setup-prompt
D:\Git\CLAUDEmdStrage\_base にある AGENTS.md と対象環境の CLAUDE_*.md を参考に、
このプロジェクト用の AGENTS.md と CLAUDE.md を作成してください。

以下を必ず整理してください。
- このプロジェクトの目的
- 実行環境
- Codex が判断すること
- Claude Code に渡す作業の条件
- 触ってよいファイルと触ってはいけないファイル
- 検証方法
- 知見を AGENTS.md / docs に残す条件

AGENTS.md と CLAUDE.md の分担

AGENTS.md に書く
なぜその設計か、何を守るか、どの作業を Claude Code に渡すか、実装後に何を見るか。
CLAUDE.md に書く
Claude Code がどう編集するか、いつ止まるか、何を検証するか、最後に何を報告するか。
docs/ に書く
長い設計説明、調査メモ、運用手順、将来また参照する技術情報。
書かない
一回限りの作業ログ、単純な文言修正、既存方針に沿った小さな差分。

最初のチェックリスト

  • プロジェクトの目的と利用者が AGENTS.md にある
  • 実行環境とデプロイ先が明確になっている
  • Claude Code に渡す条件と停止条件が分かれている
  • 機密情報や大容量ファイルが .claudeignore で除外されている
  • 自動コミットするかどうかが明記されている
