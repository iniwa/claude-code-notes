Codex CLI コマンド
Codex をローカル作業、レビュー、自動化で使うときの基本コマンドと起動オプション。

起動と実行

codexよく使う
現在のディレクトリを作業ルートとして TUI を起動する。まずはリポジトリ直下で起動し、AGENTS.md を読ませる。 
cd D:\Git\my-project
codex
codex exec "..."自動化
非対話実行。短い調査、レビュー、CI からの呼び出しに使う。標準入力も渡せる。 
codex exec "この差分をレビューして"
git diff | codex exec "変更点を要約して"
codex exec --ephemeral自動化
セッション履歴を残さず実行する。ログを残したくない一回限りの確認や、CI の短いチェック向け。 
codex exec --ephemeral "README の誤字だけ確認して"
codex app
デスクトップアプリ体験を起動する。CLI/TUI より画面で状態を見たいときに使う。

作業ディレクトリと権限

--cd / -C
起動時の作業ディレクトリを指定する。別ディレクトリからプロジェクトを指定して起動するときに使う。 
codex --cd D:\Git\ai-coding-notes
--add-dir
メインワークスペース以外の書き込み許可ディレクトリを追加する。複数指定できる。 
codex --add-dir D:\tmp
--sandbox / -s重要
サンドボックス方針を指定する。通常の編集は workspace-write、調査のみなら read-only を使う。 
codex --sandbox read-only
codex --sandbox workspace-write
--ask-for-approval / -a重要
コマンド実行前の承認方針。対話作業では on-request、完全自動化では明示的に never を検討する。 
codex -s workspace-write -a on-request

モデルと設定上書き

--model / -m
このセッションだけモデルを指定する。普段の既定値は config.toml に置き、単発の重い作業だけ CLI で上書きする。
--profile
config.toml のプロファイルを選ぶ。レビュー用、実装用、読み取り専用などを分けたいときに使う。
--config / -c
設定値を一時上書きする。永続化するほどではない試行に向いている。 
codex -c model_reasoning_effort="high"
--search
ライブ検索を使う。最新仕様、価格、リリース状況など、変わりやすい情報を扱うときに明示する。

MCP とデバッグ

codex mcp
MCP サーバー定義の追加、一覧、確認、削除に使う。設定は config.toml に保存される。 
codex mcp list
codex mcp add git -- uvx mcp-server-git
codex mcp-server連携
Codex 自体を MCP サーバーとして起動する。別の MCP クライアントから Codex をツール化したいときに使う。
codex sandbox windows
Windows サンドボックス内でコマンドがどう動くか確認する。権限まわりの再現調査に使う。 
codex sandbox windows -- powershell -NoProfile -Command "Get-Location"
codex --version
不具合報告や手順共有の前にバージョンを確認する。CLI の挙動は更新で変わるため、記録に残す。
