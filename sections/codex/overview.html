Codex 基本運用
Codex は設計を練り、設計思想の継続性を守り、Claude Code への実装ハンドオフを作る。
設計担当
要件を整理し、既存設計との関係、責務境界、採用案・不採用案を明確にする。
思想の記録
AGENTS.md や docs/ に、今後も守るべき判断を残す。
実装ハンドオフ
Claude Code が迷わず実装できるよう、目的、編集対象、制約、非目標、検証方法を整理する。
設計レビュー
実装後の差分を見て、設計思想から剥離していないか確認する。

基本ループ

Codex: 設計する
Codex: AGENTS.md に設計判断を残す
Codex: Claude Code 用 handoff を作る
Claude Code: CLAUDE.md と handoff に従って実装する
Codex: 差分を設計思想と照合する
Codex: 必要なら AGENTS.md を更新する

Codex に最初に聞くこと

prompt
これは AGENTS.md に残すべき設計判断を含む変更ですか？
含むなら、実装前に handoff と decision log を作ってください。
含まないなら、Claude Code に渡す最小の実装指示だけ作ってください。
