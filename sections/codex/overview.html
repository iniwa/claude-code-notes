Codex 基本運用
Codex は依頼を受けた時点での意思決定の場。分類、リスク判定、handoff 生成、報告レビュー、durable rule 化を一カ所で扱う。

Codex のアクション

依頼分類
変更種別（文言 / 既存調整 / 新規 / 設計変更 / ドキュメント）、影響範囲、成功条件、検証方法に分解する。
設計リスク判定
責務境界、公開 API、永続化、認証、デプロイに触れるかを判定し、高リスクなら先に AGENTS.md を更新する。
handoff 作成
docs/handoffs/YYYY-MM-DD-<task>.md に Goal / Files / Constraints / Non Goals / Verification / Expected Report を書き切る。
Claude Code 報告のレビュー
handoff 外の変更、不要なパターン追加、検証不足、公開不可情報の混入を差分から確認する。
durable rule 化の判断
今後も守るべき判断が出たときだけ AGENTS.md の Decision Log に追記する。単発の修正は記録しない。

基本ループ

Codex: 依頼を分類する
Codex: 設計リスクを判定する
Codex: そのまま実装するか handoff に落とすか決める
Codex または Claude Code: 実装・検証する
Codex: 差分を AGENTS.md と handoff に照らしてレビューする
Codex: 必要な判断だけ AGENTS.md に残す
