Codex 基本運用
Codex は設計判断を整理し、必要なら実装・検証・レビューまで一気通貫で進める。
要求整理
曖昧な依頼を、変更種別、影響範囲、成功条件、検証方法に分解する。
設計判断
既存設計との関係、責務境界、採用案・不採用案、守るべき制約を明確にする。
実装実行
変更範囲が適切に見えている場合は、Codex がそのまま編集・検証・差分整理まで進める。
実装ハンドオフ
Claude Code に渡す方が速い作業では、目的、編集対象、制約、非目標、検証方法を整理する。
設計レビュー
実装後の差分を見て、設計思想から剥離していないか確認する。
記録更新
AGENTS.md や docs/ に、今後も守るべき判断だけを残す。

基本ループ

Codex: 要求と変更種別を整理する
Codex: 設計リスクを判定する
Codex: このまま実装するか、Claude Code に渡すか決める
Codex または Claude Code: 実装・検証する
Codex: 差分を設計思想と照合する
Codex: 必要なら AGENTS.md や docs を更新する
