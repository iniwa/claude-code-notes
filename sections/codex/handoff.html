Codex で設計した内容を、Claude Code が実装できる粒度に変換する。

新規機能

新規機能では、まず設計に新しい概念を追加してよいかを確認する。既存アーキテクチャに自然に入るか、新しい責務や制約が必要かを Codex で整理してから Claude Code に渡す。

## Goal
既存の○○機能で、△△の場合の挙動を修正する。

## Change Type
Bug fix / behavior adjustment. This is not an architecture rewrite.

## Preserve
- 既存 API
- 既存 UI フロー
- 既存データ構造

## Allowed Changes
- 条件分岐の修正
- テストケース追加

- 関連する回帰テスト
