実装ハンドオフ
Codex から Claude Code に渡す唯一の入り口。docs/handoffs/YYYY-MM-DD-<task>.md をここで作る。

handoff の書き方

  • 本リポジトリの AGENTS.md と完全に同じブロック構造で書く。
  • 冒頭に「AGENTS.md、CLAUDE.md、この handoff を読むこと」「範囲外が必要なら止まって確認すること」の 2 行を入れる。
  • Files To Edit を絞る。書きすぎると Claude Code が範囲を広げる。
  • Verification はこのリポジトリで実行できるコマンドだけを書く。
  • Expected Report を毎回入れる。報告フォーマットを固定する効果がある。

ベースとなる handoff

docs/handoffs/YYYY-MM-DD-<task>.md
Read AGENTS.md, CLAUDE.md, and this handoff file before implementation.
If implementation would violate constraints or require files outside this handoff, stop and ask before editing.

例: 新規セクション追加

new-section.md
## Goal
Claude Code 用の「権限スコープ」リファレンスセクションを追加する。

## Background
~/.claude/settings.json と .claude/settings.json の優先度が頻繁に質問されるため、
独立セクションにまとめたい。

## Files To Inspect
- index.html
- js/app.js
- sections/claude/cfg-scope.html

## Files To Edit
- sections/claude/cfg-perm-scope.html  (新規)
- index.html
- js/app.js

## Constraints
- 静的 SPA を維持する。ビルドツールや依存を追加しない。
- 既存 UI パターン（.card / .card-grid / .template）だけを使う。
- nav・SECTIONS・index.html プレースホルダの 3 点更新を必ず行う。
- 公開不可情報を含めない。

## Non Goals
- 既存セクションの文言は触らない。
- CSS を追加しない。

## Verification
- git diff --check
- python -m http.server 8787 --bind 127.0.0.1
- ブラウザまたは fetch でセクションが表示されることを確認

## Expected Report
- Changed files
- Summary
- Verification results
- Blocked checks
- Design questions for Codex

例: テンプレート更新

template-update.md
## Goal
templates/claude/CLAUDE.md を本リポジトリの handoff 運用に合わせて書き直す。

## Background
公開テンプレートが古く、handoff・AGENTS.md・CLAUDE.md の三点読み込みと
報告フォーマットが反映されていない。

## Files To Inspect
- AGENTS.md
- CLAUDE.md
- templates/claude/CLAUDE.md

## Files To Edit
- templates/claude/CLAUDE.md

## Constraints
- 公開テンプレートとして汎用に保つ。個人環境固有の値は入れない。
- 「handoff、AGENTS.md、CLAUDE.md を読むこと」「allowed files の中だけで作業すること」
  「changed files / summary / verification / blocked checks / design questions を返すこと」
  の三点を必ず含める。

## Non Goals
- 他のテンプレートは変更しない。
- 新しいテンプレートは追加しない。

## Verification
- git diff --check
- テンプレート集ページで .template-loader 経由の表示が崩れていないことを確認

## Expected Report
- Changed files
- Summary
- Verification results
- Blocked checks
- Design questions for Codex

このリポジトリの検証コマンド

  • git diff --check — 末尾空白や conflict marker の検出
  • セクション配線を変えた場合: python -m http.server 8787 --bind 127.0.0.1 でローカル配信
  • HTTP 確認: Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8787/ | Select-Object -ExpandProperty StatusCode
  • ブラウザが使えない環境では blocked checks として報告する
