更新するべきケース

  • 新しい責務境界を決めた
  • 既存設計を変更した
  • 今後も守るべき制約が増えた
  • 採用しなかった案に重要な理由がある
  • Claude Code が迷いやすい判断が発生した

更新しなくてよいケース

  • 単純なバグ修正
  • 文言修正
  • CSS 微調整
  • テスト追加だけ
  • 既存方針に完全に沿った小変更
