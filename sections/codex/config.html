Codex config.toml
Codex CLI と IDE 拡張が共有する設定。ユーザー既定、プロジェクト上書き、プロファイル、MCP、機能フラグを分けて管理する。

配置と優先順位

~/.codex/config.toml
個人の既定値。普段のモデル、承認方針、MCP サーバー、通知などを置く。
.codex/config.toml
プロジェクト用の上書き。Codex が信頼済みと判断したプロジェクトで読み込まれる。
--profile
設定内のプロファイルを選ぶ。レビュー、実装、自動化などを切り替える。
-c key=value
一回限りの上書き。永続設定にする前の試行や、特定タスクだけ強める用途に使う。

基本設定例

~/.codex/config.toml
model = "gpt-5.5"
model_reasoning_effort = "medium"
sandbox_mode = "workspace-write"
approval_policy = "on-request"

[profiles.review]
model_reasoning_effort = "high"
sandbox_mode = "read-only"
approval_policy = "on-request"

[profiles.fast]
model_reasoning_effort = "low"
sandbox_mode = "workspace-write"
approval_policy = "on-request"

よく触るキー

model / model_reasoning_effort
既定モデルと思考量。普段は medium、レビューや設計判断は high、軽い文言修正は low などに分ける。
sandbox_mode
read-only、workspace-write、danger-full-access を選ぶ。通常の編集は workspace-write で十分。
approval_policy
untrusted、on-request、never を選ぶ。対話作業は on-request、自動化は実行環境を隔離して never を検討する。
log_dir
Codex のログ出力先。問題調査や長時間作業の記録を分けたいときに指定する。

MCP サーバー

MCP stdio server
[mcp_servers.git]
command = "uvx"
args = ["mcp-server-git"]
startup_timeout_sec = 10
tool_timeout_sec = 60
command / args
stdio MCP サーバーの起動コマンド。CLI の codex mcp add でも追加できる。
enabled_tools / disabled_tools
公開するツールを絞る。権限が強い MCP は allow list を優先する。
env / env_vars
MCP サーバーに渡す環境変数。秘密値をリポジトリに直接書かない。
startup_timeout_sec / tool_timeout_sec
起動待ちとツール実行待ちの上限。遅い外部ツールだけ個別に延ばす。

機能フラグと TUI

features-and-tui
[features]
shell_snapshot = true

[tui.status_line]
items = ["model", "context", "git_branch", "tokens"]
[features] は任意機能や実験機能を有効化する場所。TUI の表示、キーマップ、ステータスラインは /statusline や /keymap から調整して永続化できる。
