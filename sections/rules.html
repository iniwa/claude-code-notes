プロジェクトルール
このリファレンスを更新するときの置き場所、責任範囲、フォルダ構成の基準。

基本ルール

サイトは静的に保つ
  • ビルドステップは追加しない。
  • React / Vue / Vite / npm などは明示依頼がある場合だけ使う。
  • 既存の HTML / CSS / JS 構成を優先する。
公開できる内容だけ置く
  • API キー、認証情報、個人トークンは置かない。
  • private な運用情報はテンプレート化しない。
  • 個人環境の値は placeholder にする。
役割を混ぜない
  • サイト本文は sections/。
  • 公開テンプレートは templates/。
  • 作業指示や設計記録は docs/。

フォルダ構成

.
|-- AGENTS.md Codex 側の作業合意、役割分担、永続ルール
|-- CLAUDE.md Claude Code 側の実装ルール、検証、報告形式
|-- index.html SPA のシェル、ナビゲーション、section placeholder
|-- css/ 共通スタイル
|-- js/ section 読み込み、検索、template / markdown loader
|-- sections/ ブラウザに表示する本文
|-- templates/ 公開用の再利用テンプレート
|-- docs/ 運用文書、設計判断、handoff
|-- tools/ メンテナンス用スクリプトや補助ツール
`-- wrangler.toml デプロイ設定

docs 配下の置き場所

docs/handoffs/
Codex から Claude Code へ実装を渡す scoped handoff を置く。ファイル名は YYYY-MM-DD-<short-task>.md。
docs/decisions/
AGENTS.md に入れるには長すぎる背景、判断理由、却下案を置く。未来の作業判断に使える粒度にする。
新しい docs サブディレクトリ
既存の handoffs/ や decisions/ に入らない文書が継続的に増える場合だけ作る。

新しい section を追加するとき

  • sections/<area>/<name>.html または sections/<name>.html を追加する。
  • js/app.js の SECTIONS に id とパスを追加する。
  • index.html のナビゲーションに show('<id>') を追加する。
  • index.html の <main> に <div id="<id>" class="section"></div> を追加する。

更新判断

  • 将来の Codex / Claude Code セッションにも効く短いルールは AGENTS.md に残す。
  • Claude Code が実装時に従う具体的な実行ルールは CLAUDE.md に残す。
  • ブラウザで参照したい説明は sections/ に置く。
  • 単発の作業指示は docs/handoffs/、長期的な判断理由は docs/decisions/ に置く。
