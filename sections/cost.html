コスト・トークン最適化
Claude Code の利用コストを抑えつつ、作業効率を最大化するためのテクニック集。

モデルの使い分け

Sonnetコスパ◎
日常的なコーディング作業の大半はこれで十分。起動時に --model sonnet で指定するか、セッション中に /model sonnet で切り替え。 
# 起動時
claude --model sonnet

# セッション中（Shift+Tab 2回で fast mode トグルも可）
/model sonnet
Opus高コスト
複雑な設計判断、大規模リファクタリング、難解なバグの調査など、高い推論力が必要な場面で使う。常用は避ける。
Haiku最安
サブエージェントの model パラメータで指定すると、単純なタスク（ファイル検索、パターンマッチ等）を低コストで実行できる。 
# サブエージェントで Haiku を使う例
Agent(model: "haiku", prompt: "...")

コンテキスト管理

/compactよく使う
会話履歴を要約して圧縮する。コンテキスト使用率が 80%前後 になったら実行。自動コンパクションもあるが、手動の方がタイミングを制御できる。 
# カスタム指示付きコンパクション
/compact 実装中の認証機能の詳細を保持して
/clear
コンテキストを完全リセット。別タスクに移る時に使う。/compact は作業継続用、/clear はタスク切り替え用と使い分ける。

.claudeignore によるコンテキスト削減

Claude Code が読み込むファイルを制限することで、トークン消費を大幅に削減できる。特に大きなファイルやディレクトリを除外する。 
# .claudeignore の例
node_modules/
dist/
build/
*.min.js
*.min.css
*.map
*.lock
package-lock.json
*.bin
*.safetensors
*.gguf
効果が大きい除外対象：
package-lock.json（数万行）、*.lock、ビルド成果物、MLモデルファイル、ソースマップ。これらを除外するだけでコンテキスト消費が劇的に減る。

プロンプトの工夫

タスク分割重要
「全部実装して」より「まず認証部分だけ」の方がトークン効率が良い。大きなタスクは段階的に依頼する。1回の指示が明確なほど、やり直しが減りトータルコストが下がる。
ファイル明示参照
Claude にファイルを探させるより、@ファイル名 で直接参照する方がトークンを節約できる。 
@src/auth.py の login 関数を修正して
ワンショット活用
-p フラグでワンショット実行すると、インタラクティブモードのオーバーヘッドを避けられる。簡単な質問やコード生成に有効。 
# パイプで入力してワンショット
cat error.log | claude -p "このエラーの原因は？"
git diff --staged | claude -p "コミットメッセージを生成"
プランモード
大きなタスクの前に Shift+Tab でプランモードに切り替え。計画を確認してから実装に移ることで、方向修正によるトークン浪費を防ぐ。

サブエージェントのコスト制御

サブエージェントはメインの会話とは別にコンテキストを消費する。以下を意識する。
  • 単純な検索タスクには model: "haiku" を指定して低コスト化
  • 同じ調査をメインとサブエージェントで重複して行わない
  • サブエージェントは必要な時だけ使う — 3回以内の検索で済むなら直接 Grep/Glob を使う
  • 独立したタスクは並列実行でウォールタイムを短縮（トークンは同じだが待ち時間削減）
