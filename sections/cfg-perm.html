/config — 権限・セキュリティ
ツール使用の許可・拒否ルール。プロジェクトの .claude/settings.json に書いてgit管理するのが基本。
評価順序： deny → ask → allow の順に評価され、最初にマッチしたルールが適用される。denyを先に書けば確実にブロックできる。
permissions.defaultMode両方おすすめ
デフォルトの権限モード。"default"（毎回確認）/ "acceptEdits"（編集自動承認）/ "bypassPermissions"（全スキップ・危険） 
"permissions": { "defaultMode": "acceptEdits" }
permissions.allow両方おすすめ
確認なしで許可するツール・コマンドの一覧。ワイルドカード対応。 
"allow": [
  "Bash(npm run *)",
  "Bash(git log *)",
  "Bash(git diff *)",
  "Bash(git status)"
]
permissions.deny両方おすすめ
常に拒否するツール・コマンドの一覧。denyが最初に評価されるため確実にブロックできる。 
"deny": [
  "Bash(rm -rf *)",
  "Bash(curl *)",
  "Read(.env)",
  "Read(.env.*)"
]
filesystem.denyRead両方
読み取りを禁止するパスの一覧。秘密鍵や認証情報ファイルを保護する用途。 
"filesystem": {
  "denyRead": ["~/.ssh/*", "~/.aws/*", "~/.gnupg/*"]
}
enableAllProjectMcpServersユーザー
プロジェクトの .mcp.json を自動的に全て有効化。デフォルト false。gitリポジトリ経由の悪意あるMCPサーバー対策のため無闇に有効にしない。 
"enableAllProjectMcpServers": false
attribution.coAuthoredBy両方
gitコミットメッセージに付加されるCo-authored-by表記の制御。空文字列で非表示にできる。 
"attribution": { "coAuthoredBy": "" }
