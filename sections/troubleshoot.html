トラブルシューティング
Claude Code でよくある問題とその対処法。MCP 以外の一般的なトラブルをカバー。

コンテキスト関連

コンテキスト溢れ頻出
会話が長くなるとコンテキストウィンドウが不足し、古い情報が失われる。 
対処法:
1. /compact で圧縮（作業継続したい場合）
2. /clear で完全リセット（別タスクに移る場合）
3. .claudeignore で不要ファイルを除外
4. 大きなタスクは分割して依頼
予防策： コンテキスト使用率 80% 前後で /compact を実行する習慣をつける。ステータスバーで使用率を常時確認できる。
Claude が以前の指示を忘れる
自動コンパクション後に重要な文脈が失われることがある。 
対処法:
1. 重要な指示は CLAUDE.md に書く（毎回読み込まれる）
2. 「〜を覚えておいて」でメモリに保存
3. /compact 時にカスタム指示で保持内容を指定
   → /compact 認証機能の設計方針を保持して

権限・実行エラー

ツール実行が拒否される頻出
Claude がツールを使おうとして権限エラーになる。 
対処法:
1. /permissions で現在の権限を確認
2. .claude/settings.json の permissions.allow に追加
   例: "Bash(npm test:*)", "Write(src/**)"
3. セッション中に許可プロンプトで「常に許可」を選択
Bash コマンドがタイムアウト
デフォルトのタイムアウトは 120 秒（2分）。ビルドやテストが長い場合にタイムアウトする。 
対処法:
1. Claude に「タイムアウトを延長して」と伝える
   （最大 600 秒 / 10 分まで設定可能）
2. バックグラウンド実行を使う
3. テスト対象を絞って実行時間を短縮

MCP 関連

MCP サーバーが接続できない頻出
MCP サーバーの起動に失敗する。 
対処法:
1. claude mcp list で登録状況を確認
2. npm パッケージが存在するか確認
   → npm view @パッケージ名 version
3. npx でパッケージを直接実行してエラーを確認
4. 一度削除して再登録
   → claude mcp remove サーバー名
   → claude mcp add サーバー名 ...
よくある原因： npm パッケージ名の変更・廃止、ネットワークエラー、Node.js バージョン不一致。
MCP ツールが見つからない
MCP サーバーは接続済みだがツールが呼び出せない。 
対処法:
1. サーバーのステータスが "connected" か確認
2. Claude Code を再起動（MCPの再接続が発生）
3. permissions.allow に MCP ツールを追加
   例: "mcp__serena__find_symbol"

ファイル・Git 関連

Claude が間違ったファイルを編集
意図しないファイルが変更されてしまった。 
対処法:
1. git diff で変更内容を確認
2. git checkout -- ファイルパス で個別に戻す
3. 大きな変更の前はブランチを切る習慣をつける
4. @ファイル名 で対象を明示指定して依頼する
大きなファイルの読み込みが遅い
ログファイルやデータファイルが大きすぎてコンテキストを圧迫する。 
対処法:
1. .claudeignore に追加して除外
2. Serena の symbolic tools で必要な部分だけ読む
3. パイプで必要な部分だけ渡す
   → ! tail -100 large.log

パフォーマンス

応答が遅い
Claude の応答速度が低下している。 
対処法:
1. /compact でコンテキストを圧縮
2. Sonnet に切り替え（Opusより高速）
3. /fast でfast modeを有効化（同じモデルで高速出力）
4. 不要なMCPサーバーを削除して起動を軽量化
Claude が無限ループする
同じ操作を繰り返して進まない。 
対処法:
1. Esc で中断
2. 問題の原因を説明して方針転換を指示
3. /clear してアプローチを変えて再依頼
4. 自動化時は --max-turns で制限をかける

環境・認証

認証エラー
Claude Code の認証が切れた、または API キーが無効。 
対処法:
1. claude login で再認証
2. API キー使用の場合は ANTHROPIC_API_KEY を確認
3. Pro/Max プランの場合は claude.ai でサブスク状態を確認
